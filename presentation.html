What is SDLC?

Software Development Life Cycle

Sounds intimidating. It's just a process.

Plan → Develop → Review → Test → Deploy → Monitor

SDLC for Data Teams

P
Plan — "We need a new KPI for customer acquisition"
D
Develop — Create the measure in your own safe copy
R
Review — Team lead checks the DAX formula
T
Test — Automated checks: does the data look right?
D
Deploy — Publish to production workspace automatically
M
Monitor — Alert if dataset refresh fails overnight
SDLC is not about slowing you down.

It's about sleeping well at night.

What is Git?


Track Changes on steroids


Every change is saved forever

You always know who changed what and why

You can always go back to any previous version

Git in 4 Concepts

Repository (Repo)

Your project folder — with superpowers.
Contains all files + full history.

Commit

A saved snapshot with a message.
"Added gross margin KPI"

Branch

Your own safe copy to work on,
without affecting anyone else.

Pull Request (PR)

"I made changes — please review
before merging into production."

GitHub Actions

Automation that runs when things happen in your repo


IF someone opens a PR → THEN validate the changes
IF code merges to main → THEN deploy to Power BI
IF it's 7:00 AM → THEN check refresh status

Anatomy of a Workflow

# It's just a YAML file in your repo

name: Validate Power BI Changes   # What is it called?

on:                                      # WHEN does it run?
  pull_request:
    branches: [main]

jobs:                                    # WHAT does it do?
  validate:
    runs-on: ubuntu-latest
    steps:                            # Steps, in order
      - name: Check for .pbix files
      - name: Validate JSON files
      - name: Check documentation

Lives in .github/workflows/ — GitHub runs it automatically

Three Workflows for Power BI

1. Validate Changes on: pull_request
Block .pbix binaries • Validate JSON • Check documentation
2. Deploy to Workspace on: push to main
Authenticate • Publish via REST API • Notify team on Teams
3. Monitor Refreshes on: schedule (daily)
Check refresh status • Alert on failures • Health report

The Missing Piece


Everything we showed works great, but...


.pbix files are binary
Git can't show you what changed inside
It's like tracking changes on a ZIP file

We need Power BI content as readable text...

Enter: TMDL

Before

→ Can't review what changed
